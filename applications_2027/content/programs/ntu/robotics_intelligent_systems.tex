Migrating an ANYmal-C navigation task made me confront how commands, observations, rewards, and termination together define mobile-robot autonomy. My CR5 work raised a complementary interface problem for manipulation: visual state and policy actions must remain meaningful while the arm moves. I now want to understand how multimodal perception, task definitions, and control should change between mobile robots and manipulators. NTU's MSc in Robotics and Intelligent Systems addresses that systems-level question.

The currently published January 2027 curriculum shows how I could pursue that question if its resources remain available for my target intake. MA6214 Autonomous Mobile Robot would extend my work on command and observation definitions into multimodal fusion, planning, and motion control. MA6212 Robotics Manipulation and Advanced Control would let me compare those mobile-robot interfaces with manipulation and action execution, while MA6211 AI and Machine Learning Fundamentals for Robotics would strengthen the learning methods shared across both settings. If approved, MA6215 Robotics Projects in the coursework-only route could support a prototype that gives a mobile platform and a manipulator parallel task definitions, then examines where their sensing, action, and completion interfaces must differ. I would contribute experience across navigation migration and CR5 deployment and apply the comparison to autonomous-system R\&D.
