The capability gap I want to close lies between a model response and sustained agent behavior. I can encode robot actions and serve policy chunks, but I have not yet built an agent that orchestrates reasoning, cloud services, and physical tools across a longer task or recovers when one stage fails. The MSc in Agentic AI Systems at The Hong Kong Polytechnic University (PolyU) targets this end-to-end systems problem.

The programme's Zero-to-Physical Capstone, described as bridging software and hardware, captures the direction I want to pursue. Where the listed subjects are offered, COMP5583 Agentic Computing, Systems, and Applications would frame reasoning, tool use, and orchestration; COMP5585 Cloud Computing and Hyperscale AI Infrastructure would add the service layer; and ME5301 Embodied Robot Intelligence would carry decisions into physical sensing and control. I would use the capstone context to build an agent that decomposes a robot task, invokes learned and software tools, and responds when a service result or physical observation contradicts its plan. Hermite and CR5 would ground the representation and deployment layers of a dependable embodied agent.
